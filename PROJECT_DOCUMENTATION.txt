===============================================================================
                    ALUMNI CONNECT PROJECT DOCUMENTATION
===============================================================================

PROJECT OVERVIEW:
================
Alumni Connect is a Django-based web application that serves as a platform connecting
alumni and students of educational institutions. It facilitates networking, 
mentorship, job opportunities, events, and communication between graduates and current students.

PROJECT STRUCTURE:
=================

alumni_connect/                    # Main project directory
├── manage.py                      # Django management script
├── create_superuser.py            # Script to create admin user
├── db.sqlite3                     # SQLite database file
├── alumni_connect/                # Main project configuration
│   ├── __init__.py
│   ├── settings.py               # Django settings
│   ├── urls.py                   # Main URL routing
│   ├── wsgi.py                   # WSGI configuration
│   └── asgi.py                   # ASGI configuration
├── accounts/                     # User management app
├── events/                       # Events management app
├── mentorship/                   # Mentorship and jobs app
├── messaging/                    # Communication app
├── contributions/                # Alumni contributions app
└── templates/                    # HTML templates
    ├── accounts/                 # Account-related templates
    ├── events/                   # Event templates
    ├── mentorship/               # Mentorship templates
    ├── messaging/                # Messaging templates
    ├── contributions/            # Contribution templates
    └── base.html                 # Base template

===============================================================================
                          DETAILED APP BREAKDOWN
===============================================================================

1. ACCOUNTS APP (User Management)
=================================
PURPOSE: Handles user registration, authentication, and profile management.

MODELS:
-------
- User: Custom user model extending AbstractUser
  - Fields: username, email, first_name, last_name, role, phone, address, 
    date_of_birth, profile_picture, bio, created_at, updated_at
  - Roles: ALUMNI, STUDENT, ADMIN
  - Properties: is_alumni, is_student, is_admin_user

- AlumniProfile: Alumni-specific information
  - Fields: user, graduation_year, degree, department, current_company, 
    current_position, work_experience, skills, social media links, 
    mentoring availability

- StudentProfile: Student-specific information
  - Fields: user, enrollment_year, department, roll_number, semester, 
    skills_interest, career_goals, internship seeking

VIEWS:
------
- home: Landing page with events and announcements
- alumni_register: Alumni registration form
- student_register: Student registration form
- dashboard: User dashboard after login
- profile: View user profile
- profile_edit: Edit user profile

FORMS:
------
- AlumniRegistrationForm: Registration for alumni users
- StudentRegistrationForm: Registration for student users
- UserUpdateForm: Update user information
- ProfileUpdateForm: Update profile picture

URLS:
-----
- / : Home page
- /register/alumni/ : Alumni registration
- /register/student/ : Student registration
- /login/ : User login
- /logout/ : User logout
- /dashboard/ : User dashboard
- /profile/ : View profile
- /profile/edit/ : Edit profile

2. EVENTS APP (Event Management)
=================================
PURPOSE: Manages alumni events, reunions, workshops, and registrations.

MODELS:
-------
- Event: Event information
  - Fields: title, description, event_type, date, end_date, location, 
    is_online, online_link, organizer, max_attendees, is_published
  - Event Types: REUNION, WORKSHOP, SEMINAR, MEETUP, WEBINAR, OTHER

- EventRegistration: User registrations for events
  - Fields: event, attendee, status, registered_at
  - Status: REGISTERED, ATTENDED, CANCELLED

VIEWS:
------
- event_list: List all events
- event_detail: View event details
- event_create: Create new event
- event_update: Update existing event
- event_delete: Delete event
- register_event: Register for an event
- my_events: User's registered events

URLS:
-----
- /events/ : Event list
- /events/<id>/ : Event details
- /events/create/ : Create event
- /events/<id>/edit/ : Edit event
- /events/<id>/delete/ : Delete event
- /events/<id>/register/ : Register for event
- /events/my/ : User's events

3. MENTORSHIP APP (Mentorship & Jobs)
=======================================
PURPOSE: Facilitates mentorship between alumni and students, and job postings.

MODELS:
-------
- MentorshipRequest: Mentorship requests from students to alumni
  - Fields: mentor, mentee, subject, description, status, created_at
  - Status: PENDING, ACCEPTED, REJECTED, COMPLETED

- JobPosting: Job opportunities posted by alumni
  - Fields: posted_by, title, company, description, requirements, 
    job_type, location, salary_range, application_deadline, is_active
  - Job Types: FULL_TIME, PART_TIME, INTERNSHIP, CONTRACT

- JobApplication: Applications for job postings
  - Fields: job, applicant, cover_letter, resume, status, applied_at
  - Status: PENDING, SHORTLISTED, REJECTED, SELECTED

VIEWS:
------
- mentorship_list: List mentorship opportunities
- mentorship_request: Request mentorship
- mentorship_response: Respond to mentorship requests
- job_list: List job postings
- job_detail: View job details
- job_create: Post new job
- job_apply: Apply for job
- my_applications: User's job applications

URLS:
-----
- /mentorship/ : Mentorship list
- /mentorship/request/ : Request mentorship
- /mentorship/response/<id>/ : Respond to request
- /mentorship/jobs/ : Job listings
- /mentorship/jobs/<id>/ : Job details
- /mentorship/jobs/create/ : Post job
- /mentorship/jobs/<id>/apply/ : Apply for job
- /mentorship/applications/ : My applications

4. MESSAGING APP (Communication)
=================================
PURPOSE: Handles messaging between users and system announcements.

MODELS:
-------
- Conversation: Chat conversations between users
  - Fields: participants, subject, created_at, updated_at

- Message: Individual messages in conversations
  - Fields: conversation, sender, content, is_read, created_at

- Announcement: System-wide announcements
  - Fields: title, content, created_by, is_published, created_at

VIEWS:
------
- conversation_list: List user conversations
- conversation_detail: View conversation messages
- send_message: Send message in conversation
- create_conversation: Start new conversation
- announcement_list: View announcements
- create_announcement: Create announcement (admin)

URLS:
-----
- /messages/ : Message list
- /messages/<id>/ : Conversation details
- /messages/send/ : Send message
- /messages/new/ : New conversation
- /announcements/ : Announcements
- /announcements/create/ : Create announcement

5. CONTRIBUTIONS APP (Alumni Contributions)
===========================================
PURPOSE: Tracks alumni contributions and collects feedback.

MODELS:
-------
- Contribution: Alumni contributions tracking
  - Fields: alumnus, contribution_type, description, amount, date, 
    is_anonymous, created_at
  - Types: FINANCIAL, MENTORING, EVENT_SUPPORT, GUEST_LECTURE, OTHER

- Feedback: User feedback system
  - Fields: user, category, subject, message, status, admin_response, 
    created_at, updated_at
  - Categories: GENERAL, WEBSITE, EVENTS, MENTORSHIP, SUGGESTION
  - Status: PENDING, REVIEWED, RESPONDED

VIEWS:
------
- contribution_list: List contributions
- contribution_create: Add new contribution
- feedback_list: View feedback
- feedback_create: Submit feedback
- feedback_response: Admin response to feedback

URLS:
-----
- /contributions/ : Contributions list
- /contributions/create/ : Add contribution
- /contributions/feedback/ : Feedback list
- /contributions/feedback/create/ : Submit feedback
- /contributions/feedback/<id>/respond/ : Respond to feedback

===============================================================================
                           KEY FILES EXPLANATION
===============================================================================

manage.py
---------
- Django's command-line utility
- Used for running development server, migrations, creating superuser, etc.
- Commands: python manage.py runserver, python manage.py migrate, etc.

create_superuser.py
------------------
- Script to create initial admin user
- Creates admin user with credentials: username='admin', password='admin123'
- Run with: python create_superuser.py

alumni_connect/settings.py
---------------------------
- Main Django configuration file
- Database settings (SQLite)
- Installed apps configuration
- Authentication settings (custom user model)
- Static files and media configuration

alumni_connect/urls.py
-----------------------
- Main URL routing configuration
- Includes URLs from all apps
- Defines main project URL patterns

templates/base.html
-------------------
- Base template with common HTML structure
- Bootstrap CSS framework integration
- Navigation bar with user authentication status
- Footer with links
- Common CSS and JavaScript includes

===============================================================================
                           DATABASE RELATIONSHIPS
===============================================================================

User Model (Central):
├── AlumniProfile (One-to-One)
├── StudentProfile (One-to-One)
├── Event (Many-to-One as organizer)
├── EventRegistration (Many-to-One as attendee)
├── MentorshipRequest (Many-to-One as mentor/mentee)
├── JobPosting (Many-to-One as posted_by)
├── JobApplication (Many-to-One as applicant)
├── Conversation (Many-to-Many as participants)
├── Message (Many-to-One as sender)
├── Announcement (Many-to-One as created_by)
├── Contribution (Many-to-One as alumnus)
└── Feedback (Many-to-One as user)

===============================================================================
                           USER FLOWS
===============================================================================

Student User Flow:
1. Registration → Dashboard → Browse Events → Register for Events
2. Dashboard → Browse Jobs → Apply for Jobs
3. Dashboard → Request Mentorship → Connect with Alumni
4. Dashboard → Send Messages → Communicate with Alumni
5. Dashboard → Submit Feedback → Provide input

Alumni User Flow:
1. Registration → Dashboard → Create Profile
2. Dashboard → Create Events → Organize Reunions/Workshops
3. Dashboard → Post Jobs → Help Students
4. Dashboard → Respond to Mentorship Requests → Guide Students
5. Dashboard → Send Messages → Network
6. Dashboard → Add Contributions → Track Giving Back
7. Dashboard → View Feedback → Improve Platform

Admin User Flow:
1. Login → Dashboard → Manage All Content
2. Create Announcements → Communicate with All Users
3. Review Feedback → Respond to User Input
4. Monitor All Activities → Ensure Platform Health

===============================================================================
                           SETUP INSTRUCTIONS
===============================================================================

1. Install Python 3.11.9
2. Install Django: pip install django
3. Navigate to project directory: cd alumni_connect
4. Run migrations: python manage.py migrate
5. Create superuser: python create_superuser.py
6. Start development server: python manage.py runserver
7. Access application: http://127.0.0.1:8000

Default Admin Credentials:
- Username: admin
- Password: admin123
